# Autosclaw Setup

Guide the user through a complete installation and configuration of autosclaw.
Walk through each step interactively — verify prerequisites, build the agent image, configure environment variables, and start the manager.

## Step 1: Check Prerequisites

Verify the following are installed and available. Run each check command and report the results to the user. If any are missing, tell the user what to install before continuing.

- **Node.js 22+**: run `node --version` — must be v22 or higher
- **npm**: run `npm --version`
- **Docker**: run `docker --version` — must be running (also run `docker info` to confirm the daemon is up)

If Docker is not running, tell the user to start Docker first (`sudo systemctl start docker` on Linux, or open Docker Desktop on macOS/Windows).

## Step 2: Build the Agent Docker Image

The agent runs inside a Docker container. Build the image:

```bash
cd agent && docker build -t autosclaw-agent . && cd ..
```

Confirm the image was built successfully by running `docker images autosclaw-agent`.

If it fails, show the user the error output and help them resolve it.

## Step 3: Configure Environment Variables

Create the `.env` file in the repository root. Ask the user for values interactively using the AskUserQuestion tool, providing sensible defaults and generating secrets on the fly.

### 3a: ANTHROPIC_API_KEY (required)

Ask the user for their Anthropic API key. This is the only value that cannot be generated automatically.

Tell them: "You can get an API key from https://console.anthropic.com/settings/keys"

If they don't have one yet, pause and let them go create one.

### 3b: Admin Credentials

Ask the user to choose an admin username and password. Suggest these defaults:
- **ADMIN_USERNAME**: `admin`
- **ADMIN_PASSWORD**: Generate a secure random password using: `node -e "console.log(require('crypto').randomBytes(16).toString('base64url'))"`

Let the user accept the generated password or provide their own (minimum 6 characters).

### 3c: JWT_SECRET

Generate a secure JWT secret automatically — do not ask the user for this. Generate it with:

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

Inform the user that a JWT secret has been generated. Without this, tokens would not survive manager restarts.

### 3d: Optional Settings

Ask the user (in a single question with multiSelect) if they want to customize any of these, otherwise use defaults:

- **PORT**: `4000` (the port the dashboard and API listen on)
- **ALLOW_REGISTRATION**: `false` (whether new users can self-register)
- **LOG_LEVEL**: `info` (options: debug, info, warn, error)

Only ask follow-up questions for the options they selected.

### 3e: Write the .env File

Assemble all values and write the `.env` file to the repository root. Use this format:

```env
# Autosclaw Environment Configuration
# Generated by setup skill

# Anthropic API Key (required)
ANTHROPIC_API_KEY=<user-provided>

# Admin Account
ADMIN_USERNAME=<chosen-username>
ADMIN_PASSWORD=<chosen-or-generated-password>

# JWT Secret (auto-generated — do not change unless you want to invalidate all sessions)
JWT_SECRET=<generated-hex>

# Server Configuration
PORT=<port>
ALLOW_REGISTRATION=<true|false>
LOG_LEVEL=<level>
```

After writing, confirm the file was created and show the user a summary of their configuration (mask the API key and JWT secret, show only the first 8 characters followed by `...`).

## Step 4: Install Manager Dependencies

```bash
cd manager && npm install && cd ..
```

Report success or help debug any npm errors.

## Step 5: Build the Dashboard UI

```bash
cd manager/ui && npm install && npm run build && cd ../..
```

This compiles the React dashboard into static files served by the manager.

## Step 6: Login Information

Tell the user:

- The dashboard will be available at `http://localhost:<PORT>` (use the port from their config)
- Login with the admin credentials they just configured
- They can manage users, roles, and registration from the dashboard once logged in

## Step 7: Start the Manager

Ask the user if they want to start the manager now. If yes, run:

```bash
cd manager && npm start
```

Tell them they can stop it with Ctrl+C and restart later with the same command.

## Step 8: Install as a Systemd Service (optional)

Ask the user if they want to register autosclaw as a systemd service so it starts automatically on boot and restarts on failure. This is recommended for production/server deployments but not needed for local development.

If yes, run the install script from the repository root:

```bash
chmod +x install.sh && ./install.sh
```

This script will:
1. Verify all dependencies are available (`systemctl`, `loginctl`, `node`, `npm`, `docker`)
2. Create a systemd **user** service at `~/.config/systemd/user/autosclaw-manager.service`
3. Enable and start the service immediately
4. Enable **linger** for the current user so the service runs even without an active login session

After it succeeds, tell the user these management commands:

```
  Check status:   systemctl --user status autosclaw-manager
  View logs:      journalctl --user -u autosclaw-manager -f
  Restart:        systemctl --user restart autosclaw-manager
  Stop:           systemctl --user stop autosclaw-manager
```

If the script fails (e.g. `systemctl` not found on macOS), explain that the systemd service is Linux-only and they should use `cd manager && npm start` instead.

**Important**: If the user already started the manager manually in Step 7, tell them to stop it first (Ctrl+C) before installing the service, since both would try to bind to the same port.

## Final Summary

Once everything is complete, print a summary:

```
Autosclaw is ready!

  Dashboard:  http://localhost:<PORT>
  Username:   <admin-username>
  Password:   <shown-or-"(as configured)">

  To start:   cd manager && npm start
  To stop:    Ctrl+C
```

If the systemd service was installed, also show:

```
  Service:    autosclaw-manager (enabled, running)
  Status:     systemctl --user status autosclaw-manager
  Logs:       journalctl --user -u autosclaw-manager -f
```
